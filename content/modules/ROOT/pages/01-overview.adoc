== Your role and mission

You're a software engineer at Kaleido, and you've been assigned to build an AI-driven product recommendation system for an important business initiative.

Kaleido needs to provide intelligent and informative recommendations for a diverse product line through an intuitive user interface. Your manager has explained: "Our customers struggle to find the right products among thousands of options. We need an AI system that understands what they're looking for and provides personalized recommendations with helpful review summaries."

**Your assignment**: Build a complete AI-driven product recommender using Red Hat OpenShift AI to deliver personalized product suggestions and automated review summaries.

**The objective**: Demonstrate how OpenShift AI can power intelligent product discovery through custom ML models, feature stores, inference servers, and LLM-generated product review summaries.

== Success criteria for your mission

By the end of this workshop, you'll have practical OpenShift AI skills to address Kaleido's product recommendation requirements:

* Understand transformer model types and their use cases → **select the right AI models for specific tasks**
* Build and train custom recommendation models → **deliver personalized product suggestions**
* Use LLMs to generate review summaries → **help customers make informed buying decisions**
* Create and monitor Kubeflow pipelines → **automate AI model training and deployment**
* Deploy the Red Hat inference server → **serve AI models at scale with optimal performance**
* Manage features with OpenShift AI FEAST → **ensure consistent data across training and inference**

**Technical outcome**: Introduction to the components and technologies needed to build an AI recommendation system using enterprise-grade OpenShift AI components.

**Business benefit**: Measurable improvements in customer engagement, product discovery, and conversion rates through intelligent AI-driven recommendations.

== Kaleido's product recommendation challenges

**The situation**: Kaleido needs to transform how customers discover products across their diverse catalog, improving engagement and conversion rates.

**Project timeline**: Deliver a working AI recommendation system that demonstrates the value of personalized product discovery and automated review summarization.

**Current challenges** (operational pain points):

* **Poor product discovery**: Customers struggle to find relevant products in a large catalog → high bounce rates and abandoned searches
* **Information overload**: Hundreds of reviews per product overwhelm customers → delayed purchase decisions
* **Generic recommendations**: Simple rule-based suggestions miss customer preferences → low click-through rates
* **Manual curation**: Product teams manually create recommendation lists → doesn't scale and lacks personalization

**The opportunity**: Red Hat OpenShift AI provides the complete platform needed to build intelligent product recommendations with custom models, feature management, and scalable inference.

**Technical perspective**: "OpenShift AI offers the integrated tooling we need -- from Jupyter workbenches for model development to Kubeflow Pipelines for automation to KServe for production serving -- but I need to evaluate how these components work together for our use case."

== Your vision of success

If the AI recommendation system proves effective for Kaleido's use case, here's the potential technical improvements:

**Immediate improvements** (first release):

* **Personalized recommendations**: Custom two-tower model delivers product suggestions based on user preferences → 30-40% improvement in click-through rates
* **Automated review summaries**: LLM-generated summaries distill key insights from hundreds of reviews → faster customer decisions

**Strategic benefits** (longer-term):

* **Scalable AI infrastructure**: OpenShift AI platform supports multiple AI use cases → reduced time-to-market for new AI features
* **Consistent feature management**: FEAST feature store eliminates training-serving skew → improved model accuracy in production
* **Automated ML workflows**: Kubeflow Pipelines enable continuous model improvement → sustained recommendation quality

**Success metric**: A demonstrable AI recommendation system that improves product discovery, accelerates buying decisions, and establishes a foundation for future AI initiatives at Kaleido.

== Common questions

**"What types of AI models will we use for recommendations?"**
→ Module 1 explains the four key models: BGE text embeddings, CLIP image embeddings, Llama or Qwen for review summaries, and a custom two-tower recommender model

**"Do we need to use the above specific models?"**
→ No. There are many models to choose from and factors you must consider. We'll review some of these factors in Module 1, but for now keep in mind that though you can use Llama to generate review summaries, you can explore among many other options. Start by exploring models on the Red Hat repository on Huggingface here (https://huggingface.co/RedHatAI). If you are running this showroom on the Red Hat Developer Platform, the application is deployed with a remote Qwen3 14-B model running on Red Hat's Model-as-a-Service platform (MaaS) with Intel's Gaudi accelerator. If you deploy this quickstart on your cluster, you must provide the endpoint and credentials for the endpoint of your inference server.

**"How do we prevent training-serving skew in production?"**
→ Module 2 demonstrates using the FEAST feature store to maintain consistent user and product features across training and inference

**"How do we serve AI models at scale?"**
→ Module 3 shows deploying models with the Red Hat inference server and KServe for high-performance, concurrent request handling

**"Can we automate model training and deployment?"**
→ Module 2 covers building Kubeflow Pipelines that orchestrate the complete training workflow from data loading to embedding generation
