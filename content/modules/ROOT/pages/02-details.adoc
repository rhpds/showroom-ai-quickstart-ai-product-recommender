== Timing and schedule

=== Full workshop (1 hour)

* **Module 1**: Architecture and AI models (15-20 minutes)
* **Module 2**: Feature store and training pipeline (20-25 minutes)

This workshop is designed for self-paced learning. You can:

* Complete all modules in sequence for the full experience
* Focus on specific modules based on your interests
* Return to review concepts and re-run exercises

== Module breakdown

=== Module 1: Architecture and AI models (15-20 min)

**Focus**: Understanding the AI-driven product recommender architecture and model selection

**Topics**:

* Product recommender system architecture
* Transformer model types and use cases
* Text embeddings with BGE model
* Image embeddings with CLIP model
* LLMs for generative tasks
* Two-tower recommendation model architecture

**Hands-on activities**:

* Explore the deployed product recommender application
* Examine AI models and their roles in the system
* Review FEAST feature store configuration

=== Module 2: Feature store and training pipeline (20-25 min)

**Focus**: Building and managing AI training workflows

**Topics**:

* FEAST feature store concepts and benefits
* Kubeflow Pipelines architecture
* Two-tower model training workflow
* Pipeline component development
* Model training and batch scoring
* Embedding generation and storage

**Hands-on activities**:

* Explore FEAST feature views and services
* Review Kubeflow Pipeline definitions
* Monitor pipeline execution and logs
* Examine trained model artifacts

=== Module 3: Inference server and LLM integration (15-20 min)

**Focus**: Deploying models for production inference

**Topics**:

* Red Hat inference server capabilities
* KServe serving runtime
* Model deployment and scaling
* LLM integration for review summarization
* Context window management
* Production inference patterns

**Hands-on activities**:

* Test product recommendations
* Generate review summaries with LLM
* Interact with the OpenShift Pipeline Server

== Troubleshooting guide

=== Common access issues

**Problem**: "Cannot access OpenShift console"
→ **Solution**: Verify you're using the correct URL from `{openshift_console_url}`. Clear browser cache and try again.

**Problem**: "Login credentials not working"
→ **Solution**: Ensure you're using `{user}` and `{password}` exactly as provided. Check for extra spaces.

**Problem**: "Showroom interface not loading"
→ **Solution**: Refresh the browser. If issue persists, try a different browser or incognito mode.

**Problem**: "Cannot see deployed components"
→ **Solution**: Verify you're in the correct OpenShift project/namespace. Check the project selector in the console.
